# LucentChamber — 小说创作辅助工具 项目大纲

> 本文档记录项目讨论过程中的所有决策、待定事项和变更历史，作为唯一的项目大纲使用。

---

## 一、项目概述

为小说作者提供一套可视化的创作辅助工具，支持：
- 设定世界观中的时间节点与历史大事件
- 管理人物设定档案（含类型化自定义字段）
- 定义人物之间的关系网络（节点连线图）
- 管理故事地图/场景设定（含图片）
- 结合本地 AI Agent（DeepSeek v4 Pro）根据设定自动填充故事细节
- 小说编辑器（纯文本 + 独立插图面板）

---

## 二、技术选型（已确定）

```
Electron 28 + Vue 3.5 + Vite 5 + TypeScript
├── 架构: Electron IPC（主进程直连渲染进程，无 HTTP 层）
├── 前端 UI: Element Plus 2.13
├── 状态管理: Pinia 2
├── 路由: Vue Router 4
├── 数据库: better-sqlite3（账号系统）
├── 数据文件: JSON（人物/事件/关系/地图）+ TXT（小说正文）
├── BUILD: Vite 5
└── 图片管理: 用户选择本地图片 → 程序复制到项目本地目录统一管理
```

| 项 | 决定 |
|---|------|
| 语言 | 纯 JS/TS（Python 不参与） |
| 交付形式 | Electron 打包为独立 EXE |
| 多用户 | 本地账号系统，每个用户数据隔离 |
| AI 集成 | 暂不实现，后续通过 opencode 等 Agent 工具调用 DeepSeek v4 Pro |

## 三、数据目录结构（已确定）

```
~/LucentChamber/
├── users.db                          # SQLite（账号密码）
└── workspaces/
    └── 《小说名称》/
        ├── project.json              # 项目元信息（名称、创建时间等）
        ├── characters/
        │   ├── 张三.json             # 每个角色独立文件，含详细设定
        │   └── 李四.json
        ├── timeline.json             # 时间节点
        ├── events.json               # 历史大事件（含关联角色）
        ├── relationships.json        # 人物关系（预定义 + 自定义）
        ├── maps.json                 # 地图/场景设定
        ├── novel.txt                 # 小说正文（纯文本）
        └── images/                   # 插图/地图图片本地副本
            ├── characters/
            ├── maps/
            └── illustrations/
```

## 三、功能模块（6 个页面 + 左侧侧边栏导航）

| 模块 | 路由 | 数据源 | 说明 |
|------|------|--------|------|
| 时间线 | `/timeline` | `timeline.json` | 横向时间轴 |
| 大事件 | `/events` | `events.json` | 事件列表，可关联角色 |
| 人物设定 | `/characters` | `characters/*.json` | 公共字段 + 类型化自定义字段 |
| 人物关系 | `/relationships` | `relationships.json` | 节点连线图 |
| 地图 | `/maps` | `maps.json` + 图片 | 地形为主，不标记角色位置 |
| 编辑器 | `/editor` | `novel.txt` | 纯文本编辑 + 独立插图面板 |

## 四、人物设定字段设计（已确定）

### 公共字段
| 字段 | 类型 |
|------|------|
| 姓名 | text |
| 性别 | select（男/女/其他） |
| 年龄 | number |
| 性格 | text |
| 外貌 | textarea |
| 背景故事 | textarea |
| 头像 | image |

### 自定义字段
用户可在角色编辑界面对每个角色添加/删除自定义字段。
支持类型：`text` / `number` / `date` / `textarea` / `select`（下拉选项） / `image`
